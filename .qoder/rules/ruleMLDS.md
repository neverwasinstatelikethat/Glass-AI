Для всех ML/DS компонентов придерживайтесь следующих стандартов:
- Используйте типизацию (`typing` модуль) для всех функций
- Документируйте функции в формате Google Style
- Сохраняйте модели через MLflow с логированием параметров и метрик
- Для временных рядов всегда делайте train/test split по времени, а не случайно
- LSTM архитектуры должны включать:
  * 2-3 слоя LSTM (128-256 units)
  * Dropout слой (0.2-0.3) для предотвращения переобучения
  * Batch normalization между слоями
- Прогнозные модели должны возвращать не только предсказание, но и доверительные интервалы
- Все гиперпараметры должны быть вынесены в конфигурационные файлы
- Используйте wandb или mlflow для трекинга экспериментов
